Executive summary

This milestone report is written for the Coursera Johns Hopkins Data Science Specialization capstone project.

The Capstone Project involves developing a predictive model of text starting with a really large, unstructured database of the English language.This report display the progress on the development of the predictive model. It contains the following sections: 1. Load and prepare data 2. Summary statistics on the text datasets 3. Further steps in building the model

Analysis description

Around the world, people are spending an increasing amount of time on their mobile devices for email, social networking, banking and a whole range of other activities. But typing on mobile devices can be a serious pain. SwiftKey, our corporate partner in this capstone, builds a smart keyboard that makes it easier for people to type on their mobile devices. One cornerstone of their smart keyboard is predictive text models. When someone types:

I went to the

the keyboard presents three options for what the next word might be. For example, the three words might be gym, store, restaurant.

This reports show the underlying datasources and progress of the development of the predictive model.

Module 2: Load and prepare data and determine subsample

In this module, the text data is loaded and analyzed for generating descriptive statistics on the available data.As the data is very large, a subsample of the data is generated from the 3 different input files. Next to the source text fieles, additional files are loaded such as a file with profanity language to extract this from the source data. In the data cleaning stage the profanity language is removed and a standard format of the words (e.g. lower, no leading/trailing white spaces) is determined.

# Create empty stats dataframe
stats_list <- list()

# Loop through filenames to derive text data
for (text_filename in text_filenames){
        
        # Determine paths
        load_text_path <- paste(curr_dir, language, text_filename, sep = '/')
        save_text_path <- paste(curr_dir, language, paste0('sample_', text_filename), sep = '/')
        
        # Read text data and derive summary statistics
        text_data <- readLines(load_text_path, encoding = "UTF-8", skipNul=TRUE)
        
        # Get file size
        file_size <- round((file.info(load_text_path)$size / 1024 ^ 2),2)
        # Number of lines
        n_lines <- length(text_data)
        # Number words in file
        n_words <- sum(sapply(gregexpr("\\S+", text_data), length))
        # Collect stats in array and append to dataframe
        text_stats <- c(text_filename, file_size, n_lines, n_words)
        stats_list[[text_filename]] <- text_stats
        
        # Check if sample file already exists within directory
        directory_files <- list.files(paste(curr_dir, language, sep = '/'))
        sample_existence_flag <- ifelse(length(directory_files[grepl(paste0('sample_', text_filename), 
                                                               directory_files, fixed = TRUE)]) > 0, TRUE, FALSE)
        
        # If sample file does not exist create and save
        if(sample_existence_flag == FALSE){ 

                # Read a subsample of lines from the core datasource
                sample_text_data <- sample_lines(load_text_path, (length(load_text_path)/100)*1) # this give 1% of lines 
        
                # Save sample text data
                write.table(sample_text_data, file = save_text_path, row.names = FALSE, sep = ';')
                print(paste('Sample file saved', text_filename, sep = ' '))
                } else {
                        paste('Sample file not saved', text_filename, sep = ' ')
                }
}

Module 4: Further steps in building the model

This concludes the exploratory analysis of the datasets which will be used for the predictive model. In the next step, the predictive model will be created and integrated into a Shiny app for word prediction.

The ngram frequency dataframes are used to calculate the probability of the next word occuring. The input string will be tokenized and the words before the last word in the n-gram will be isolated and matched against the frequency data frames to identify the ngram with the higest probability and substract the predictid word from this ngram.

In order to let a user interact with the model, a shiny application will be build. In this application, a user can provided a line of text for which the most likely options of the next word will be provided.
